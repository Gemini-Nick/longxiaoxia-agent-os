# UI/UX Optimization Checklist

## Product Boundary

- 产品定位：个人金融领域超级工作交易台。
- 主模式保持不变：策略 / 回测 / 执行 / 微信 / 插件。
- 优化范围是设计、交互、比例、信息密度和体验，不改变功能定位。
- 设计语言参考 Vibe Island 的 SwiftUI 质感：克制、紧凑、原生、清晰。
- 硬性风格：极简。默认首屏只保留当前可操作内容，解释性文本、诊断细节、路径和长日志进入详情层。
- 前端审查基线：frontend-skill / frontend-design / figma-implement-design / web-design-guidelines / react-best-practices / playwright / webapp-testing / canvas-design / brand-guidelines / vercel-deploy-claimable。

## User Feedback To Address

- 全局视觉、配色、字体、交互必须统一，不能只是换字体。
- 所有页面的信息冗余度过高，需要更精简。
- 页面和交互尺寸有挤压，比例不协调。
- 点击路径要逐个验证，不能只看页面加载。
- 执行页过度设计，默认信息和功能太多。
- 插件页过度设计，默认信息和功能太多。
- 中文界面下不能混入一堆英文；英文界面也要保持一致。
- 报错、提示和说明都要短句化，不能大段解释。
- 微信绑定需要更精炼。
- 微信的功能映射、入站路由、会话管理定位不清晰，不能像介绍页，要像可操作工作台。
- 左侧功能栏目比例不对，需要重新调整。
- 左侧底部状态栏需要更精简。
- 右侧详情抽屉内容、比例和布局不规范，存在重叠和错误细节。
- 需要用 Computer Use 逐页检查策略 / 回测 / 执行 / 微信 / 插件。
- 每个页面的关键按钮都要用视觉和点击验证。
- 所有优化必须服务于极简、统一、低负担的工作台体验。
- 每个按钮必须有明确目的；无意义、重复或只解释概念的按钮/入口要删减或合并。

## Acceptance Checks

- 五个主模式首屏不挤压、不重叠。
- 中文模式下可见状态、提示、按钮和错误文案保持中文；代码、路径、URL、命令可保留原样。
- 默认首屏只展示当前任务所需信息，高级诊断和底层路径不抢占主视图。
- 详情抽屉覆盖式打开，不挤压主工作区；长路径和 JSON 不撑破布局。
- 左侧 rail 比例稳定，导航和状态区都保持轻量。
- 点击主导航、刷新、分段控件、路由测试、插件折叠、详情打开等关键按钮无 renderer error。
